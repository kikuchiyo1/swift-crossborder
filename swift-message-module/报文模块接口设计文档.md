# 报文模块接口设计文档

## 1. 提供给外部模块的接口（报文模块提供）

### 1.1 发报接口
- **接口路径**: `POST /api/message/outbound`
- **功能**: 接收外部模块（清算/支付）的报文，发送到SWIFT网络
- **请求参数**:
  ```json
  {
    "xml": "完整的报文XML原文"
  }
  ```
- **返回参数**:
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "msgId": "PACS008-xxx"
    }
  }
  ```
- **错误码**:
  - `400`: xml为空或无法提取BizMsgIdr
  - `409`: 报文已存在（重复提交）

### 1.2 查询接口
- **接口路径**: `GET /api/message/{msgId}`
- **功能**: 外外部模块查询报文详情
- **请求参数**: 
  - `{msgId}`: 报文ID
- **返回参数**:
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "msgId": "PACS008-xxx",
      "msgType": "pacs.008",
      "rawContent": "报文XML原文"
    }
  }
  ```
- **错误码**:
  - `404`: 报文不存在

## 2. 调用外部模块的接口（报文模块调用）

### 2.1 清算模块回调接口
- **接口路径**: `POST /api/v1/swift/payment/settle`
- **功能**: 收到SWIFT报文后回调清算模块
- **请求参数**:
  ```json
  {
    "msgId": "PACS008-xxx",
    "msgType": "pacs.008"  // 或 "pacs.002"
  }
  ```
- **返回参数**:
  ```json
  {
    "returnCode": "SUCCESS",  // 或其他业务状态码
    "errMessage": "success"
  }
  ```

## 4. 表设计

### 4.1 收到报文表 (inbound_message)
```sql
CREATE TABLE inbound_message (
    id              BIGINT       AUTO_INCREMENT PRIMARY KEY,
    msg_id          VARCHAR(64)  NOT NULL COMMENT 'BizMsgIdr 去重键',
    msg_type        VARCHAR(20)  NOT NULL COMMENT 'pacs.008 / pacs.002 / ACK / NACK',
    sender_bic      VARCHAR(11)  COMMENT '发送方BIC（AppHdr.Fr）',
    receiver_bic    VARCHAR(11)  COMMENT '接收方BIC（AppHdr.To）',
    raw_content     MEDIUMTEXT   NOT NULL COMMENT '报文XML原文',
    status          VARCHAR(30)  NOT NULL DEFAULT 'PENDING_ROUTING',
    retry_count     INT          NOT NULL DEFAULT 0 COMMENT '重试次数，上限3',
    created_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE INDEX uk_msg_id (msg_id),
    INDEX idx_status (status)
);
```

### 4.2 待发送报文表 (outbound_message)
```sql
CREATE TABLE outbound_message (
    id              BIGINT       AUTO_INCREMENT PRIMARY KEY,
    msg_id          VARCHAR(64)  NOT NULL COMMENT 'BizMsgIdr 去重键',
    msg_type        VARCHAR(20)  NOT NULL COMMENT 'pacs.008 / pacs.002 / ACK / NACK',
    sender_bic      VARCHAR(11)  COMMENT '发送方BIC（AppHdr.Fr）',
    receiver_bic    VARCHAR(11)  COMMENT '接收方BIC（AppHdr.To）',
    raw_content     MEDIUMTEXT   NOT NULL COMMENT '报文XML原文',
    status          VARCHAR(30)  NOT NULL DEFAULT 'PENDING_ROUTING',
    retry_count     INT          NOT NULL DEFAULT 0 COMMENT '重试次数，上限3',
    created_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE INDEX uk_msg_id (msg_id),
    INDEX idx_status (status)
);
```

## 5. SDK解析字段

### 5.1 关键字段提取
- **报文ID**: `BizMsgIdr` (用于去重和关联)
- **报文类型**: 通过 `FIToFICstmrCdtTrf` (pacs.008) / `FIToFIPmtStsRpt` (pacs.002) / `<Ack>` / `<Nack>` 识别
- **发送方BIC**: `AppHdr.Fr.BICFI`
- **接收方BIC**: `AppHdr.To.BICFI`

### 5.2 状态常量
- `PENDING_ROUTING`: 待路由
- `IN_PROGRESS`: 处理中
- `DELIVERED`: 已送达
- `FAILED`: 失败

## 6. 调用示例

### 6.1 调用发报接口示例
```javascript
// 使用 axios 调用发报接口
const response = await axios.post('http://your-server:8080/api/message/outbound', {
  xml: '完整的报文XML内容'
});

if (response.data.code === 200) {
  console.log('发报成功，报文ID:', response.data.data.msgId);
} else {
  console.log('发报失败:', response.data.message);
}
```

### 6.2 调用查询接口示例
```javascript
// 使用 axios 调用查询接口
const response = await axios.get('http://your-server:8080/api/message/PACS008-xxx');

if (response.data.code === 200) {
  const { msgId, msgType, rawContent } = response.data.data;
  console.log('查询成功:');
  console.log('  msgId:', msgId);
  console.log('  msgType:', msgType);
  console.log('  rawContent:', rawContent);
} else {
  console.log('查询失败:', response.data.message);
}
```
